■ アプリ起動（Run）

【概要】

アプリケーションやファイル、フォルダ、URLを開きます。
AutoHotkeyで最も使用頻度の高い命令の一つです。

【基本構文】

Run "実行したいもの"

【サンプル】

メモ帳を起動

Run "notepad.exe"

【応用例1】

電卓を起動

Run "calc.exe"

【応用例2】

フォルダを開く

Run "C:\Users\User\Documents"

【応用例3】

Webサイトを開く

Run "https://www.google.com"

【応用例4】

ファイルを開く

Run "C:\Work\sample.xlsx"

Excelがインストールされていれば自動でExcelが起動します。

【解説】

Runには次のようなものを指定できます。

指定するもの 例
実行ファイル notepad.exe
フルパス C:\Tool\App.exe
フォルダ C:\work
URL https://...
ファイル sample.xlsx

【注意点】

スペースを含むパスは必ずダブルクォーテーションで囲みます。

Run "C:\Program Files\App\App.exe"

■ テキスト入力（SendText）

【概要】

文字列をそのまま入力します。
日本語・英数字・記号などの入力に適しています。

【基本構文】

SendText "入力する文字列"

【サンプル】

SendText "こんにちは"

【応用例1】

住所入力

SendText "神奈川県横浜市…"

【応用例2】

メールアドレス入力

SendText "sample@example.com"

【応用例3】

改行を含めて入力（`nは改行コード）

SendText "1行目`n2行目"

【解説】

SendTextは文字列をそのまま送信します。

SendText "^c"
↓
^c
と入力されます。

【注意点】

ショートカットキーを送りたい場合はSendを使用します。

■ キー入力（Send）

【概要】

キーボード操作そのものを送信します。
ショートカットキーの送信にも使用します。

【基本構文】

Send "送信するキー"

【サンプル】

Enterキー

Send "{Enter}"

【応用例1】

Ctrl + C

Send "^c"

【応用例2】

Ctrl + Shift + Esc

Send "^+{Esc}"

【応用例3】

Windows + R

Send "#r"

【応用例4】

Tabキーを5回押す

Loop 5
{
　Send "{Tab}"
}

【解説】

キー 記述
Enter {Enter}
Tab {Tab}
Esc {Esc}
Space {Space}
Delete {Delete}
Backspace {Backspace}
Up {Up}
Down {Down}
Left {Left}
Right {Right}

【注意点】

文字入力だけならSendTextの方が安全です。
キー操作を送る場合のみSendを使用しましょう。

■ クリップボード

【概要】

コピーした内容を取得・変更できます。
定型文入力やデータ加工によく利用します。

【基本構文】

コピー内容取得

text := A_Clipboard

コピー内容変更

A_Clipboard := "Hello"

【サンプル】

クリップボードを表示

MsgBox A_Clipboard

【応用例1】

コピー内容を大文字にする

A_Clipboard := StrUpper(A_Clipboard)

【応用例2】

コピー内容を小文字にする

A_Clipboard := StrLower(A_Clipboard)

【応用例3】

コピーして表示

Send "^c"
ClipWait
MsgBox A_Clipboard

【解説】

ClipWaitはコピーが完了するまで待機します。
これを書かないと

Send "^c"
MsgBox A_Clipboard

では古い内容を取得してしまう場合があります。

【注意点】

クリップボードを書き換えると元の内容は失われます。
必要なら事前に保存

old := A_Clipboard

■ 待機（Sleep）

【概要】

指定した時間だけ処理を停止します。
アプリの起動待ちなどによく使用します。

【基本構文】

Sleep 1000

1000 = 1秒

【サンプル】

Run "notepad.exe"
Sleep 1000
SendText "Hello"

【応用例1】

0.5秒待機

sleep 500

【応用例2】

3秒待機

sleep 3000

【応用例3】

キー入力の間隔を空ける

Send "A"
Sleep 100
Send "B"
Sleep 100
Send "C"

【解説】

単位はミリ秒です。

値 時間
100 0.1秒
500 0.5秒
1000 1秒
5000 5秒

【注意点】

必要以上に長いsleepは処理を遅くする原因になります。

■ メッセージ表示（MsgBox）

【概要】

メッセージを表示します。
デバッグや確認によく使用します。

【基本構文】

MsgBox "表示内容"

【サンプル】

MsgBox "処理を終了しました"

【応用例1】

変数を表示

name := "田中"
MsgBox name

【応用例2】

複数行表示

MsgBox "1行目`n2行目"

【応用例3】

クリップボード内容確認

MsgBox A_Clipboard

【解説】

デバッグ時には

MsgBox "ここまで来た"

を挟むだけでも、処理がどこまで実行されているか確認できます。

【注意点】

MsgBoxは閉じるまで処理が停止します。
処理を止めたくない通知にはTrayTipなど別の方法もありますが、まずはMsgBoxを使いこなせるようになるとよいでしょう。

第2章のまとめ

この章では、AutoHotKeyでよく使う基本的な処理を学びました。

処理 用途
Run アプリ・ファイル・フォルダ・URLを開く
SendText 文字列をそのまま入力する
Send キー操作やショートカットを送信する
A_Clipboard クリップボードの内容を取得・変更する
Sleep 一定時間待機する
MsgBox メッセージを表示する

これらはほぼすべてのAutoHotKeyスクリプトで使用される基本機能です。
