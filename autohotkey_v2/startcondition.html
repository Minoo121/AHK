■キーボードからキーを入力した時（Hotkey）

【概要】

特定のキーやキーの組み合わせが押されたときに処理を開始します。
最もよく使用される起動条件です。

【基本構文】

キー::{
　処理
}

または（AutoHotkey v2で推奨）

キー::{
　処理
}

【サンプル】

F1キーを押すとメモ帳を起動

F1::{
　 Run "notepad.exe"
}

【応用例1】

Ctrl + Shift + A

^+a::{
　MsgBox "実行されました"
}

【応用例2】

Windows + Eで任意のフォルダを開く

#e::{
　Run "C:\Work"
}

【応用例3】

Alt + QでGoogleを開く

!q::{
　Run "https://www.google.com"
}

【解説】

記号 キー
^ Ctrl
! Alt
+ Shift
# Windowsキー

複数組み合わせることもできます。

^!+F1
↓
Ctrl + Alt + Shift + F1

【注意点】

キー名はAutoHotkeyが定義している名前を使用します。

例）
  • Enter
  • Space
  • Esc
  • Tab
  • Delete
  • Home
  • End
  • PgUp
  • PgDn

■キーワードを入力した時（Hotstring）

【概要】

特定の文字列が入力された瞬間に処理を開始します。
単語の置換だけではなく、プログラムを実行することもできます。

【基本構文】

::文字列::
{
　 処理
}

【サンプル】

::abc::{
　MsgBox "abcが入力されました"
}

【応用例1】

メールアドレスを自動入力

::mail::{
　SendText "sample@example.com"
}

【応用例2】

住所を入力

::addr::{
　SendText "神奈川県○○市..."
}

【応用例3】

::today::{
　SendText FormatTime(, "yyyy/MM/dd")
}

【解説】

Hotstringは通常、

  • スペース
  • Enter
  • Tab

などの終了キーを押したタイミングで発動します。

即座に実行したい場合

:*:abc::{
　MsgBox "入力されました"
}

*オプションを付けることで、終了キーを待たずに反応します。

【注意点】

大量のHotstringを登録すると管理しづらくなるため、用途ごとにファイルを分けることをおすすめします。

■ 特定ソフトが起動した時（タイマー監視）

【概要】

AutoHotkeyには「アプリが起動した瞬間」を直接検知する機能はありません。
そのため、一定間隔でプロセスを監視する方法が一般的です。

【基本構文】

SetTimer CheckApp, 1000

CheckApp() {
　; 処理
}

1000は1秒ごとを意味します。

【サンプル】

メモ帳が起動したら通知

global started := false

SetTimer CheckNotepad, 1000

CheckNotepad(){
　if WinExist("ahk_exe notepad.exe") {
　　if !started {
　　started := true
　　MsgBox "メモ帳が起動しました"
　　}
　} else {
　　started := false
　}
}

【応用例1】

起動したら別ソフトも起動

Run "calc.exe"

【応用例2】

起動したらウィンドウサイズ変更

WinMove 0, 0, 1200, 700, "ahk_exe notepad.exe"

【応用例3】

WinActivate "ahk_exe notepad.exe"
Sleep 500
SendText "起動しました"

【解説】

よく使用する検索条件

  • 実行ファイル名

    ahk_exe

  • ウィンドウクラス

    ahk_class

  • プロセスID

    ahk_pid

【注意点】

監視間隔を短くしすぎるとCPU使用率が上がります。

一般的には

500ms～1000ms

程度が推奨されます。

■ 特定ソフトがアクティブな間だけ有効

【概要】

特定ソフトが最前面にある時だけショートカットを有効にします。
ゲームや業務アプリで非常によく使われます。

【基本構文】

#HotIf WinActive("条件")
キー::{
}

#HotIf

最後の

#HotIf

で条件を解除します。

【サンプル】

メモ帳でのみF1が有効

#HotIf WinActive("ahk_exe notepad.exe")

F1::{
　 MsgBox "メモ帳です"
}

#HotIf

【応用例1】

Excelだけで有効

#HotIf WinActive("ahk_exe EXCEL.EXE")

F2::{
　MsgBox "Excel専用"
}

#HotIf

【応用例2】

ゲーム専用ショートカット

#HotIf WinActive("ahk_exe Game.exe")

F12::{
　SendText "こんにちは"
}
#HotIf

【応用例3】

ブラウザだけで有効

#HotIf WinActive("ahk_exe chrome.exe")

^1::{
　 MsgBox "Chrome専用"
}

#HotIf

【解説】

WinActive()にはさまざまな条件を指定できます。

  • 実行ファイル名で判定

    WinActive("ahk_exe notepad.exe")

  • ウィンドウタイトルで判定

    WinActive("ahk_class Notepad")

  • ウィンドウクラスで判定

    WinActive("ahk_class Notepad")

用途に応じて使い分けますが、
実行ファイル名（ahk_exe）による判定が最も安定しておりおすすめです。

【注意点】

#HotIfを解除し忘れると、その後に定義したすべてのホットキーにも条件が適用されてしまいます。
そのため、条件付きホットキーの定義が終わったら、必ず次のように記述する癖をつけましょう。

第1章のまとめ

この章では、AutoHotkeyの基本となる4つの起動条件を学びました。

  • Hotkey：キー入力をきっかけに処理を実行
  • Hotstring：文字列入力をきっかけに処理を実行
  • タイマー監視：特定ソフトの起動を検知して処理を実行
  • #HotIf：特定ソフトがアクティブな間だけホットキーを有効化
