■ 変数

【概要】

変数は、値を一時的に保存するための入れ物です。
文字列・数値・計算結果などを保存できます。

【基本構文】

変数名 := 値

【サンプル】

name := "田中"
MsgBox name
↓
田中

【応用例1】

数値を保存する

count := 10
MsgBox count

【応用例2】

計算する

print := 100
tax := 10

total := price + tax

MsgBox total
↓
110

【応用例3】

文字列を結合する

lastName := "田中"
firstName := "太郎"

fullName := lastName . firstName

MsgBox fullName
↓
田中太郎

【解説】

AutoHotKey v2では代入に:=を使用します。
比較に使用する=や==と混同しなように注意しましょう。

【注意点】

変数名には意味が分かる名前を付けることをおすすめします。

■ if / else（条件分岐）

【概要】

条件によって実行する処理を変更します。

【基本構文】

if 条件 {
　処理
}else {
　処理
}

【サンプル】

age := 20

if age >= 18 {
　MsgBox "成人です"
}else{
　MsgBox "未成年です"
}

【応用例1】

メモ帳が起動してるか判定

if WinExist("ahk_exe notepad.exe"){
　MsgBox "起動しています"
}

【応用例2】

クリップボードが空か判定

if A_Clipboard = ""{
　MsgBox "何もコピーされていません"
}

【応用例3】

複数条件

score := 85

if score >= 80{
　MsgBox "合格"
}else{
　MsgBox "不合格"
}

【解説】

よく使う比較演算子

演算子 意味
== 等しい
!= 等しくない
> より大きい
<< /td> より小さい
>= 以上
<=< /td> 以下

【注意点】

ブロックが1行でも{}を付けることをおすすめします。

■ switch（複数分岐）

【概要】

値によって複数の処理を切り替えます。
ifが増えてきたらswitchを検討しましょう。

【基本構文】

switch 値{
case 値1:

　処理
case 値2:

　処理
default:
　処理
}

【サンプル】

day := "土"

switch day {
case "月":
　MsgBox "平日"

case "土":
　MsgBox "休日"

default:
　MsgBox "その他"
}

【応用例】

アプリ名ごとに処理を変える

app := "chrome"

switch app{
case "chrome":
　Run "chrome.exe"

case "notepad":
　Run "notepad.exe"

dafault:
　MsgBox "未対応"
}

【解説】

同じ変数を何度も比較する場合は、ifよりもswitchの方が読みやすくなります。

【注意点】

条件式(age >= 20など)には向いていません。そのような場合はifを使用しましょう。

■ Loop（回数指定の繰り返し）

【概要】

指定した回数だけ処理を繰り返します。

【基本構文】

Loop 回数 {
　処理
}

【サンプル】

5回メッセージを表示

Loop 5{
　MsgBox "Hello"
}

【応用例1】

10回Tabキー

Loop 10 {
　Send "{Tab}"
}

【応用例2】

番号付きで表示

Loop 5 {
　MsgBox A_Index
}

【解説】

A_Indexは現在の繰り返し回数です。

結果
1
2
3
4
5

【注意点】

Loopの回数を非常に大きくすると、処理に時間がかかります。

■ while（条件付き繰り返し）

【概要】

条件が真の間だけ繰り返します。

【基本構文】

while 条件 {
　処理
}

【サンプル】

count := 1

while count <= 5 {
　MsgBox count
　count++
}

【応用例】

アプリが終了するまで待つ

while WinExist("ahk_exe notepad.exe") {
　Sleep 1000
}

【解説】

count++は

count := count + 1

と同じ意味です。

【注意点】

条件が変化しないと無限ループになります。

■ for（コレクションの繰り返し）

【概要】

配列やオブジェクトの内容を順番に処理します。

【基本構文】

for 変数 in 配列 {
　処理
}

【サンプル】

fruits := ["りんご", "みかん", "ぶどう"]

for fruit in fruits {
　MsgBox fruit
}

【応用例】

番号付きで表示

fruits := ["りんご", "みかん", "ぶどう"]

for index, fruit in fruits{
　MsgBox index ": " fruit
}
↓
1: りんご
2: みかん
3: ぶどう

【解説】

forは配列やオブジェクトを扱うときに使用します。
単純な回数指定ならLoopの方がシンプルです。

【注意点】

配列が空の場合は一度も実行されません。

■ 関数

【概要】

処理をひとまとめにいて、何度でも呼び出せるようにします。

【基本構文】

関数名(){
　処理
}

【サンプル】

Hello(){
　MsgBox "こんにちは"
}

Hello()

【応用例1】

名前を受け取る

Hello(name){
　MsgBox "こんにちは " name
}

Hello("田中")

【応用例2】

合計を返す

Add(a, b){
　return a + b
}

result := Add(5, 10)

MsgBox result

【解説】

returnは、関数の結果を呼び出し元へ返します。

【注意点】

同じ処理を複数回書くなら、関数にまとめることで保守性が向上します。

第3章のまとめ

この章では、スクリプトを柔軟に制御するための基本構文を学びました。

構文 用途
if/else 値を保存する
switch 複数の選択肢から処理を選ぶ
Loop 指定回数だけ繰り返す
while 条件が真の間だけ繰り返す
for 配列やオブジェクトを順番に処理する
関数 処理をまとめて再利用する

実務での使い分け

最後に、実際にスクリプト作成での使い分けを覚えておくと便利です。

  • 決まった回数だけ繰り返す → Loop
  • 条件が満たされるまで繰り返す → while
  • 複数の選択肢を分岐する → switch
  • 条件に応じて処理を変える → if / else
  • 同じ処理を何度も使う → 関数
  • 複数のデータを順番に処理する → for

この考え方を身につけると、どの構文を選べばよいか判断しやすくなります。
